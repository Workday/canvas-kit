# Box

[![LABS: Beta](https://img.shields.io/badge/LABS-beta-orange)](https://github.com/Workday/canvas-kit/tree/master/modules/labs-react/README.md)
This component is work in progress and currently in pre-release.

`Box` is a primitive component that provides a common, ergonomic API around Canvas design tokens. It
is highly flexible, and its primary purpose is to build other components.

## Style Props

While `Box` supports `as`, `ref`, and all semantic element props, style props are at its heart.
Style props are special design-system-aware props that allow you to use Canvas design tokens
efficiently as you build components. Style props are better understood by their categories, which
are listed below. Each category has a more detailed explanation in the following subsections.

### Categories

- [Border](#border)
- [Color](#color)
- [Depth](#depth)
- [Layout](#layout)
- [Position](#position)
- [Space](#space)

### Border

Border style props allow you to adjust the border styles of components. Some props, such as
`borderRadius` and `borderColor`, are connected to our design tokens. This makes it really simple to
use Canvas values as you build UI. There are five types of border props: `BorderShorthandProps`,
`BorderColorProps`, `BorderRadiusProps`, `BorderStyleProps`, and `BorderWidthProps`.

#### Border Shorthand Props

These props map to CSS border shorthand properties:

| Prop Name      | Style           |
| -------------- | --------------- |
| `border`       | `border`        |
| `borderTop`    | `border-top`    |
| `borderRight`  | `border-Right`  |
| `borderBottom` | `border-bottom` |
| `borderLeft`   | `border-left`   |

---

### Space

Space style props allow you to adjust the margin and padding of components. There are two types of
space props: `LogicalSpaceProps` and `StandardSpaceProps`.

#### Logical Space Props

Logical space props are based on
[CSS Logical Properties](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_Logical_Properties/Basic_concepts).
They support bidirectionality (LTR and RTL) and will automatically flip the value when a change in
direction is detected. For example: `marginInlineStart="16px"` would set `margin-left: 16px;` for
LTR and switch to `margin-right: 16px;` for RTL. You'll want to opt for these properties over
standard space props, such as `marginLeft` and `marginRight` when you need bidrectional styles.

Here's a quick reference for these props and how they map to LTR and RTL styles:

| Logical Space Prop   | LTR Style       | RTL Style       |
| -------------------- | --------------- | --------------- |
| `marginInlineStart`  | `margin-left`   | `margin-right`  |
| `marginInlineEnd`    | `padding-right` | `padding-left`  |
| `paddingInlineStart` | `padding-left`  | `padding-right` |
| `paddingInlineEnd`   | `padding-right` | `padding-left`  |

---

#### Standard Space Props

Standard space props work the same as logical space props, but have no bidirectional support. This
will really only affect `marginLeft`, `marginRight`, `paddingLeft`, and `paddingRight`.

Here's a quick reference for these props and their styles:

| Standard Space Prop | Styles                             |
| ------------------- | ---------------------------------- |
| `margin`            | `margin`                           |
| `marginX`           | `margin-left` and `margin-right`   |
| `marginY`           | `margin-top` and `margin-bottom`   |
| `marginTop`         | `margin-top`                       |
| `marginRight`       | `margin-right`                     |
| `marginBottom`      | `margin-bottom`                    |
| `marginLeft`        | `margin-left`                      |
| `padding`           | `padding`                          |
| `paddingX`          | `padding-left` and `padding-right` |
| `paddingY`          | `padding-top` and `padding-bottom` |
| `paddingTop`        | `padding-top`                      |
| `paddingRight`      | `padding-right`                    |
| `paddingBottom`     | `padding-bottom`                   |
| `paddingLeft`       | `padding-left`                     |
