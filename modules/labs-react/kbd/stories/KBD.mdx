# KBD

`KBD` is a compound component used to visually represent keyboard input, such as keys or keyboard
shortcuts.

## Installation

```sh
yarn add @workday/canvas-kit-labs-react
```

## Usage

### Basic Example

`KBD` is a container that wraps one or more `KBD.Item` components. Each `KBD.Item` represents a
single keyboard key.

> **Important:** When displaying a group of keyboard keys (such as <kbd>Ctrl</kbd> + <kbd>C</kbd>),
> **do not** place that combination as a text into a single `KBD.Item` (for example,
> `<KBD.Item>Ctrl + C</KBD.Item>`). Doing so will cause issues with RTL (right-to-left) positioning
> and keyboard shortcut display. Instead, wrap each key in its own `KBD.Item` and group them inside
> a `KBD` container. This ensures correct rendering and layout in both LTR and RTL environments.

<ExampleCodeBlock code={Basic} />

### Dynamic Items

`KBD` is built on the [Collection API](/get-started/for-developers/guides/collection-api/). Instead
of rendering `KBD.Item` components statically, you can pass an array of strings to the `items` prop
and provide a render prop as the children. Each string represents the label of a single keyboard
key.

<ExampleCodeBlock code={Items} />

### RTL Example

You can also use the `KBD` component in right-to-left (RTL) layouts. This is helpful for languages
that are read from right to left.

<ExampleCodeBlock code={RTL} />

### Size

The `KBD` component supports different sizes. You can adjust the size using the `size` prop with
options like `small`, `medium`, and `large` to suit various UI needs. The `size` is shared with each
`KBD.Item` through the model.

<ExampleCodeBlock code={Size} />

### Variant

The `KBD` component supports different variants through the `variant` prop. Use `default` for the
standard style or `plain` for a more minimal appearance. The `variant` is shared with each
`KBD.Item` through the model.

<ExampleCodeBlock code={Variant} />

## Model

`KBD` uses a `useKBDModel` to track the keyboard keys (`items`) and to share the `size` and
`variant` with each `KBD.Item`. If you need direct access to the model's state, you can create it
yourself with `useKBDModel` and pass it to `KBD` via the `model` prop.

```tsx
const model = useKBDModel({
  size: 'large',
  variant: 'plain',
  items: ['⌘', 'C'],
});

<KBD model={model}>{item => <KBD.Item>{item}</KBD.Item>}</KBD>;
```
